# Product Roadmap

This roadmap keeps JAVA2GO focused on conversion quality, explainability, and developer trust.

## Current Product Thesis

JAVA2GO helps developers migrate Java projects to Go by producing understandable, reviewable Go output plus clear warnings, limitations, and follow-up guidance.

## MVP Goals

- Convert a focused subset of common Java project structures.
- Produce idiomatic Go for supported cases.
- Clearly label warning, partial, unsupported, and error outcomes.
- Show users what was converted, what needs review, and what could not be converted.
- Maintain a small but reliable regression suite.

## Beta Goals

- Expand Java feature coverage based on real project fixtures.
- Improve conversion reporting and user-facing explanations.
- Add stronger project-level conversion workflow support.
- Build a documented known-limits page and examples.

## Later Goals

- Support larger real-world migration workflows.
- Add richer diff, review, and repair loops.
- Explore commercial packaging around migration quality, auditability, and saved engineering time.

## Not Now

- Claiming full Java compatibility.
- Supporting every Java framework equally.
- Prioritizing visual polish over conversion accuracy.
- Building broad enterprise features before core conversion trust is proven.
